README intro:
Hello_Eliza is a deliberately constrained conversational website. It responds to text input using pattern matching and simple state—not machine learning—by design.

The project explores how hidden mechanics shape behavior. Intermittent reinforcement drives engagement (and addiction). Conversational interfaces invite trust (and misplaced belief). Systems that obscure how they work can manipulate; systems that expose how they work can teach. **Hello_Eliza uses the same techniques—and then shows you the seams.**

This is not a critique from the outside. ELIZA fooled therapists in 1966 with a lookup table. The same pattern-matching logic powers slot machines, engagement algorithms, and the uncanny valley where correlation is mistaken for understanding. The difference here is not sophistication, but consent.

Essays will follow on interface design, human-in-the-loop systems, and the gap between *"I don't understand this"* and *"this was built so I wouldn't."*

## Pp below the title in the readme.md
This is a website that asks for your attention before it gives you content.
Inspired by Zork, ELIZA, and the old internet, Hello_Eliza is part text adventure, part blog, part meditation on how interfaces shape cognition. Type something, and it responds — first with ASCII art, then with conversation, then (if you find the words) with essays about technology and attention.
The codebase is also self-documenting: tagged comments explain design decisions and can be surfaced as dialogue. The machine explains itself when asked. Built with ASP.NET Core MVC, vanilla JavaScript, and the belief that curiosity should be rewarded.


The professional signal is there (full-stack, clean architecture, deployed), but wrapped in something that says "I think about what I build." That's rarer and more interesting than another CRUD app.

Create, Read, Update, Delete — the four basic database operations.
"CRUD app" is shorthand for standard business software that just shuffles data in and out of a database. Login, forms, tables, save buttons. Functional, necessary, boring.
It's slightly dismissive — "just another CRUD app" means nothing interesting is happening architecturally or creatively. Hello_Eliza is pointedly not that.
